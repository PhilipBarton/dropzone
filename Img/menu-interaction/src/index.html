Generally, if you have a container, and then you'd like to rock a full-width (viewport) image, some markup would be required to make it happen. You'd have to close the container, add your 100% image, then reopen the container.

That's all well and good. But, this becomes a bit of an issue when creating blog post templates and layouts in CMS's like Wordpress. Sure, you can write some custom functions to wrap images with the required markup, then add another media upload option in the editor's toolbar, but honestly, it's a pain in the ass.

Hello Viewport Units

No worries playas. There's a better way. With viewport units, namely viewport width (vw), you create this full-width grid breaking pattern with a few lines of painless css.

The Run Down

SO let's say your container is 60% in width. That leaves 40%, or 20% on each side of the centered container. Now, just apply a width and max-width of 100vw to your grid breaking element, with a margin-left of 20vw. Boom. We're done son! 

/*-------------------------------------   
--Row/Container
---------------------------------------*/    
.row{  
  width: 90%;
  margin-left: auto;
  margin-right: auto;
  
  @media (min-width: $mq-small) {
    max-width:	80%;
  }
 	@media (min-width: $mq-large) {
   	max-width: 60%
 	}
}
/*-------------------------------------   
--Grid Breaker
---------------------------------------*/
.gridbreak{
  width: 100vw;
  max-width: 100vw;
  margin-left: -5vw;
  @media (min-width: $mq-small) {
    margin-left: -10vw;
  }
  @media (min-width: $mq-large) {
    margin-left: -20vw;
  }  
}

No weird hacks, to overflows to cut. No custom functions in your CMS of choice. No custom Wordpress QTags bullshit. Just a nice grid-breaking full-width image. Just check these full-width mfers out right here ya'll:

Media Query Hotness

Of course, your container is gonna increase/decrease width as you resize. So, just make sure to use the same simple-ass maths at each breakpoint. Check the CSS tab if I'm not splain' it well.
